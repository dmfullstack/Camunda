Start Via Tasklist

  • Log in to camunda Tasklist (username: demo; passwort: demo)
  • Click button 'start process' and choose process (depending on modeling tool, in case of doubt choose 'invoice receipt')
  • Select local PDF file
  • Click on 'start process'
  • In the tasklist of the demo user you will see the task 'assign approver'
  • Click on 'work on task' to work on task
  • Possibilities in form:
    • Form: enter data and finish task
    • Invoice: show PDF file
    • Delegate to: comment the task and delegate it to another user
    • Process: to view BPMN diagramm including information about the current state and the already completed tasks
  • Click through the whole process this way

SVN is required

In the last task a subversion system (SVN) is called in order to archive the PDF file. This only works if in the class "PDFTOSVNDelegate" an existing SVN has been provided and the credetials have been entered correctly.

Even if there is no SVN, the process can be run through without a problem until the last user task before the SVN call.

Human Workflow Management with JSF

The process application requires camunda Tasklist and does not bring a tasklist on its own.

camunda userguide

Almost all the assignments in the user tasks (attribute "assignee" in user task) point at the user name "kermit". The only exception is "approve invoice". Here, a variable "approver" is set in the attribute "assignee", which is set in the JSF form "assignApprover" via a select box.

There is one JSF form per user task, which itself is is embedded into an template (WEB-INF/templates/template.xhtml). Thereby we can define a central basis layout for the process application in which all forms are integrated consistently.

In the template, a variant of twitter bootstrap is referenced, adapted to camunda Tasklist, in order to align the design of the forms to the desing of the camunda tasklist. This approach can also be used to implement your own corporate design.

camunda userguide
Service Calls with Java Delegates

At the end of the process the PDF file is filed in a subversion (SVN). In BPMN this is a service task which itself points to the Java class "PDFToSVNDelegate". Here all the process variables are read and the SVN system is called.

As you can see, via this very simple approach the whole range of Java can be used. One could also easily implement a link to other systems within this delegate.
CDI integration

The CDI integration of the camunda engine is used quite often in this application, e.g. in order to start aprocess instance out of the servlet, which can be called up from the start form ("StartByUpload").

The example shows how quick and easy even very individual requirements for your process applications can be realized through CDI integration.

Visualize the running process

At every given point in the process one can see at which point in the process one stands and which steps have been alreday taken and who has completed the tasks.

This function demonstrates the process diagramm API, with which one can view an image file of the deployed process and also gets a 'coordinate system" which is based on the layout information of the exported BPMN 2.0-XML. By this means the function can be combined with any BPMN 2.0-conform tool.

Built in this, with the help of CCS and Javascript ( (JQuery, Twitter Bootstrap and other functions) the named features are implementet in this example.

The example shall show how easy and flexible your own and solution-specific runtime information on the process can be inserted into an on business-side understandable process diagramm.
